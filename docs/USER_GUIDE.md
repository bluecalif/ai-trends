## 서비스 개요

AI Trend Monitor는 다음과 같은 기능을 제공합니다:

- **RSS 피드 수집**: 10개 주요 AI 뉴스 소스에서 최신 기사 수집
- **AI 기반 분류**: IPTC/IAB 표준 분류 + 커스텀 AI 태그로 기사 분류
- **사건 그룹화**: 유사한 기사를 그룹화하여 타임라인으로 구성
- **인물 추적**: 주요 AI 인물의 활동을 타임라인으로 추적
- **북마크**: 관심 있는 기사를 저장하고 태그로 관리

## 주요 기능

### 홈 페이지

홈 페이지(`/`)에서는 모든 분야의 최신 아이템을 확인할 수 있습니다.

**주요 기능**:
- 최신 아이템 목록 표시
- 분야별 필터링 (Research, Industry, Infra, Policy, Funding)
- 커스텀 태그 필터링 (agents, world_models, non_transformer 등)
- 페이지네이션
- 날짜 범위 필터링

### 아이템 카드

각 아이템 카드는 다음 정보를 표시합니다:

- **제목**: 기사 제목 (클릭 시 원문 링크)
- **요약**: 기사 요약 (RSS description)
- **태그**: 커스텀 AI 태그 (agents, world_models 등)
- **출처**: RSS 소스 이름
- **발행 시간**: 상대 시간 표시 (예: "2 hours ago")
- **사건 링크**: 동일 사건이 여러 개인 경우 "View story" 링크

## 분야별 아이템 조회

### 분야 탭

상단의 분야 탭을 클릭하여 특정 분야의 아이템만 필터링할 수 있습니다:

- **Research**: 연구 및 학술 논문
- **Industry**: 산업 뉴스 및 제품 발표
- **Infra**: 인프라 및 하드웨어
- **Policy**: 정책 및 규제
- **Funding**: 투자 및 펀딩

### 필터링 옵션

각 분야 페이지에서 다음 필터를 사용할 수 있습니다:

1. **커스텀 태그 필터**: 
   - agents
   - world_models
   - non_transformer
   - neuro_symbolic
   - foundational_models
   - inference_infra

2. **날짜 범위 필터**: 
   - 시작 날짜와 종료 날짜를 선택하여 특정 기간의 아이템만 조회

3. **정렬 옵션**:
   - 발행 시간 기준 오름차순/내림차순
   - 생성 시간 기준 정렬

## 사건 타임라인 보기

### 사건 그룹

유사한 기사들은 자동으로 그룹화되어 하나의 "사건"으로 표시됩니다.

### 타임라인 보기

1. 아이템 카드에서 "View story" 링크 클릭
2. 또는 `/story/[groupId]` URL로 직접 접근

**타임라인 페이지 기능**:
- 시간순으로 정렬된 관련 기사 목록
- 각 기사의 발행 시간과 출처 표시
- 원문 링크로 이동 가능

## 인물 페이지 사용법

### 인물 목록

`/persons` 페이지에서 추적 중인 인물 목록을 확인할 수 있습니다.

**초기 추적 인물** (5명):
- Yann LeCun (Meta, JEPA)
- Andrej Karpathy (NanoChat, Eureka Labs)
- David Luan (Agentic AI, Amazon Nova)
- Llion Jones (Sakana AI)
- AUI/Apollo-1 (Neuro-symbolic)

### 인물 상세 페이지

인물을 클릭하면 상세 페이지(`/persons/[id]`)로 이동합니다.

**표시되는 정보**:
- 인물 이름 및 소개
- 타임라인: 인물과 관련된 최신 이벤트
- 관계 그래프: 인물-기술-기관-사건 관계 (향후 구현 예정)

**이벤트 타입**:
- paper: 논문 발표
- product: 제품 출시
- investment: 투자/인수
- organization: 조직 이동
- research_direction: 연구 방향 발표

## 북마크 기능

### 북마크 생성

1. 아이템 카드에서 "저장" 버튼 클릭
2. 북마크 모달에서 다음 정보 입력:
   - **제목**: 북마크 제목 (기본값: 아이템 제목)
   - **태그**: 북마크 태그 (여러 개 가능)
   - **메모**: 추가 메모 (선택사항)

### 북마크 관리

`/saved` 페이지에서 저장한 북마크를 관리할 수 있습니다.

**기능**:
- 북마크 목록 조회
- 태그별 필터링
- 검색 기능
- 북마크 삭제
- 기존 태그 자동완성

### 북마크 태그

태그를 사용하여 북마크를 분류하고 나중에 쉽게 찾을 수 있습니다.

**태그 사용 팁**:
- 의미 있는 태그 사용 (예: "important", "follow-up", "research")
- 여러 태그 조합 가능
- 기존 태그는 자동완성으로 재사용

## 설정 페이지

`/settings` 페이지에서 소스와 워치 규칙을 관리할 수 있습니다.

### 소스 관리

**기능**:
- RSS 소스 목록 조회
- 새 소스 추가
- 소스 수정/삭제
- 소스 활성화/비활성화 토글

**초기 소스** (10개):
- TechCrunch
- VentureBeat – AI
- MarkTechPost
- WIRED (All)
- The Verge (All)
- IEEE Spectrum – AI
- AITimes
- arXiv – cs.AI
- OpenAI News
- The Keyword (Google DeepMind filtered)

### 워치 규칙 관리

워치 규칙은 특정 인물이나 키워드와 관련된 기사를 자동으로 매칭합니다.

**기능**:
- 워치 규칙 목록 조회
- 새 규칙 추가
- 규칙 수정/삭제
- Form 모드와 JSON 모드 전환

**규칙 구성**:
- **Label**: 규칙 이름 (인물 이름으로 자동 설정 가능)
- **Required Keywords**: 필수 키워드 (모두 포함되어야 함)
- **Optional Keywords**: 선택적 키워드 (하나라도 포함되면 됨)
- **Include Rules**: 포함 규칙 (OR 조건)
- **Exclude Rules**: 제외 규칙
- **Priority**: 우선순위 (높을수록 우선)

**규칙 예시**:
```json
{
  "label": "Yann LeCun",
  "required_keywords": ["JEPA", "Meta"],
  "optional_keywords": ["LeCun"],
  "include_rules": ["JEPA", "I-JEPA", "V-JEPA", "Meta"],
  "exclude_rules": [],
  "priority": 10
}
```

## 문제 해결

### 아이템이 표시되지 않음

**가능한 원인**:
1. RSS 수집이 아직 실행되지 않음
2. 필터 조건이 너무 제한적
3. 데이터베이스 연결 문제

**해결 방법**:
1. 설정 페이지에서 소스가 활성화되어 있는지 확인
2. 필터를 초기화하고 다시 시도
3. 페이지 새로고침

### 타임라인이 비어있음

**가능한 원인**:
1. 인물과 관련된 기사가 아직 수집되지 않음
2. 워치 규칙이 제대로 설정되지 않음

**해결 방법**:
1. 설정 페이지에서 워치 규칙 확인
2. RSS 수집이 정상적으로 실행되는지 확인
3. 시간이 지나면 자동으로 업데이트됨

### 북마크가 저장되지 않음

**가능한 원인**:
1. 네트워크 연결 문제
2. 서버 오류

**해결 방법**:
1. 네트워크 연결 확인
2. 페이지 새로고침 후 재시도
3. 브라우저 콘솔에서 에러 메시지 확인

## 자주 묻는 질문 (FAQ)

### Q: RSS 수집 주기는 어떻게 되나요?

A: 일반 소스는 20분마다, arXiv는 하루 2회(00:00, 12:00 EST) 수집됩니다.

### Q: 새로운 RSS 소스를 추가할 수 있나요?

A: 네, 설정 페이지에서 새 소스를 추가할 수 있습니다.

### Q: 인물을 추가로 추적할 수 있나요?

A: 네, 설정 페이지에서 인물을 추가하고 워치 규칙을 설정할 수 있습니다.

### Q: 데이터는 얼마나 오래 보관되나요?

A: 현재는 제한이 없지만, 향후 데이터 보관 정책이 추가될 수 있습니다.

### Q: 모바일에서 사용할 수 있나요?

A: 네, 반응형 디자인으로 모바일에서도 사용 가능합니다.
